# Skill Trigger Notes

Use this file to track whether skills trigger naturally and whether their descriptions are specific enough.

## What To Record

- the prompt used
- which skill appeared to trigger
- whether the triggered skill was the right one
- whether a narrower or broader description would help

## Example Questions

- Did the Docker task use the Alpine-specific skill or only the generic Docker skill?
- Did the Angular task correctly prefer the Signals reviewer?
- Did a Rust testing task stay focused on tests, or drift into broad implementation?
- Did GitLab rules debugging trigger the narrow rules skill or the broader pipeline skill?
